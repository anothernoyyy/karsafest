Panduan Penggunaan KarsaFest

Cara menguji tiketing berbasis Merkle Tree secara offline & online

Konsep Dasar

Tiket Anda divalidasi menggunakan Merkle Tree. Panitia hanya menyimpan 1 nilai ringkasan data (Merkle Root) di blockchain, dan tiket Anda diverifikasi menggunakan data pembuktian jalur (Merkle Proof) tanpa pernah menunjukkan rahasia tiket Anda ke panitia.

1 Metode A: Uji Coba Offline (Simulasi Lokal)

Metode ini gratis, instan, dan tidak memerlukan wallet atau token asli.

  1. Lihat kolom kiri "Daftar Tiket Aktif" (sudah ada 3 tiket bawaan).
  2. Klik tombol "Hitung Merkle Root & Tree" di bagian bawah kolom kiri.
  3. Klik salah satu tiket dalam daftar (misal tiket VIP pertama) untuk memunculkan detail kodenya.
  4. Klik **"Masukkan ke Input Gate Check-in"** untuk menyalin bukti ke gerbang kanan.
  5. Klik **"Verifikasi Lokal (Simulasi)"** di kolom kanan. Gerbang akan menyala hijau ACCESS GRANTED (VALID)!

2 Metode B: Uji Coba On-Chain (Blockchain & MetaMask)

Metode ini menguji interaksi langsung dengan smart contract asli.

  1. Hubungkan dompet digital Anda menggunakan tombol **"Connect Wallet"** di Navbar kanan atas.
  2. Deploy file KarsaTix.sol di Remix IDE dan salin alamatnya. Tempelkan ke panel admin di bawah (Panitia Console) lalu simpan.
  3. Tambahkan tiket baru atau gunakan yang ada, klik **"Hitung Merkle Root & Tree"** di kolom kiri, lalu salin kode root (`0x...`) yang tampil.
  4. Di **Panitia Console**, masukkan root baru tersebut ke kolom **"Update Root On-Chain"** lalu klik **"Update Root"**. Setujui transaksi MetaMask.
  5. Masukkan data tiket terpilih ke Gate, lalu klik **"Verifikasi Smart Contract"**. Hasilnya akan divalidasi langsung dari blockchain secara gasless (gratis)!
